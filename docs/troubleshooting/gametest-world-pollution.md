# Игровые тесты (`runGameTestServer`) падают «без причины»

## Симптом

- `.\gradlew.bat runGameTestServer` падает: `Process ... finished with non-zero exit value 2`.
- В `run/logs/latest.log` падают тесты, которые **раньше проходили** и которые вы **не меняли**.
- В начале лога — много накопленных деревень: `... world runtime ready with N Village records`
  (N большое: десятки).
- Часто рядом есть ошибки `Synt Citizen binding rejected: DUPLICATE_ENTITY`.
- Ассерты падают на нуждах: например `hunger should be critical after decay`
  (голод почему-то НЕ упал).

## Причина

`runGameTestServer` использует **один и тот же постоянный мир** `run/world` и **не очищает**
его между запусками. Деревни, жители и сущности из прошлых прогонов накапливаются. Со временем:

1. новый тестовый житель не может привязаться к своей сущности (`DUPLICATE_ENTITY`,
   на занятой позиции) и остаётся **не-ACTIVE**;
2. не-ACTIVE житель **пропускается** в `simulateCitizen` → его нужды (голод/отдых) **не убывают**;
3. тест, ждущий «голод стал критическим после спада», видит голод ~1000 и падает.

То есть падает **не логика мода**, а замусоренный тестовый мир.

> Как отличить от настоящего бага: если «позитивный» тест (ожидает, что действие произошло)
> проходит, а падают только «негативные»/старые — почти наверняка это мусор в мире, а не код.

## Решение — сброс тестового мира

Удалить накопленный мир (одиночные миры пользователя в `run/saves` НЕ трогаем):

```bash
# Git Bash
rm -rf run/world
```

```powershell
# PowerShell
Remove-Item -Recurse -Force run\world
```

Мир пересоздастся чистым при следующем `runGameTestServer`, и тесты снова проходят.

## Профилактика

- Сбрасывайте `run/world`, если тесты начали падать на ассертах про нужды/сущности,
  а код вы не трогали.
- Пишите ассерты **по конкретной сущности** (по `citizenId`), никогда по глобальным счётчикам
  (`citizenCount() == 1` и т.п.) — глобальные счётчики ломаются от накопленного состояния.
- Тесты, которые кладут блоки/жителей, не должны полагаться на чистоту мира.
