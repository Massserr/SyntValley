# Решения частых / долгих проблем (dev troubleshooting)

Эта папка — «память» по повторяющимся проблемам разработки SyntValley. Когда что-то
ломается уже не в первый раз, решение записывается сюда, чтобы не искать заново
(в том числе после сброса контекста у ассистента).

## Указатель

- [Игровые тесты падают «без причины» → грязный тестовый мир](gametest-world-pollution.md)
- [Изменение жителя не сохраняется → ревизия выросла на 2 за один тик](citizen-revision-single-increment.md)
- [Gradle не запускается в среде Claude → проверка чистой Java харнессом](gradle-in-claude-env.md)

## Формат записи

Каждый файл строится по одной схеме, чтобы решение читалось за 10 секунд:

1. **Симптом** — как проблема выглядит снаружи (текст ошибки, поведение).
2. **Причина** — что на самом деле сломалось.
3. **Решение** — точные команды/шаги.
4. **Профилактика** — как не наступить снова.

Если симптом не совпал ни с одним файлом — разберитесь и **заведите новый файл** здесь.
